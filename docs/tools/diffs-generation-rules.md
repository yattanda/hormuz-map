# index_html_diffs.md 形式ルール・スケルトン

> **位置づけ：このファイルは、かつて CLAUDE.md に記載されていた `index_html_diffs.md` の形式ルール・必須スケルトン・セルフチェックを、CLAUDE.md の常時コンテキスト外に切り出した補助ドキュメントである。**
>
> - `docs/tools/index_html_diffs.md` は Claude.ai がデイリー更新作業のために生成する handoff ファイルであり、このファイルとは別物。移動・変更はしていない。
> - このドキュメントは Claude.ai が自動参照するルールファイルではなく、diffs.md の形式や運用メモをまとめた参照用資料である。
> - Claude Code の常時コンテキストには載せない。

---

## なぜこのルールが必要か

`diffs.md` を「記憶と慣習」で生成すると、セクションが漏れる。
連日の更新漏れ（速報インシデント・シナリオ確率補足バナー等）の根本原因はここにある。
以下のスケルトンを毎回の生成起点として使うことで漏れをゼロにする。

---

## Claude.ai の絶対手順

`diffs.md` を生成する際は、以下を必ず守ること：

1. 下記「必須セクション対応表」を確認する
2. 全11セクションのスケルトンを先に展開する（内容は空欄のまま）
3. 各スケルトンに情報を埋める
4. 全11セクションが埋まっていることを確認してからファイルを出力する
5. ダウンロードリンクを提示する前に下記「出力前セルフチェック」を実施する

## ⚠️ 特に漏れやすい2セクション（要注意）

### [S03] 速報インシデント が漏れる理由

- HTML の位置が MAP と STATS の後（ページ後半）にあるため
- TICKER や情勢カードより目立たない
- 「どうせニュースカラムと重複する」と思い込みやすい

→ **対策：必ず diffs.md の S03 スロットを先に作り、そこに書く**

### [S06] シナリオ確率補足バナー が漏れる理由

- CLAUDE.md に「シナリオ確率の数値は自動同期・手動更新不要」と書かれているため「補足バナーのテキストも不要」と誤解しやすい
- シナリオ4本（[S07]）と同じ `<!-- SCENARIOS -->` 内にあるため「[S07] を更新すれば [S06] も含まれる」と誤認しやすい

→ **対策：補足バナーは S06 として独立セクション扱いにし、必ず別途書く**

## 出力前セルフチェック（ダウンロードリンク提示前に必ず実施）

以下を全て確認してからダウンロードさせること。1つでも NO ならその場で追記すること。

```
[ ] S01 ヘッダー ― 日付YYYY年MM月DD日 HH:MM JSTが入っているか
[ ] S02 TICKER ― 本日の主要ニュースが反映されているか
[ ] S03 速報インシデント ― 存在するか・本日の新インシデントが入っているか ← 最重要確認
[ ] S04 情勢カード3枚 ― 3枚とも日付・数値・出典が更新されているか
[ ] S05 COUNTDOWN ― フェーズラベルが現状に合っているか
[ ] S06 シナリオ確率補足バナー ― 存在するか・日付と矢印テキストが更新されているか ← 最重要確認
[ ] S07 シナリオ4本 ― タイトルと本文が最新情勢を反映しているか
[ ] S08 シナリオフッター ― 次の焦点5点が最新か
[ ] S09 30秒カラム ― 3行サマリーとバッジ5枚が全て更新されているか
[ ] S10 news_data.jsonメモ ― アーカイブ注意事項が記載されているか
[ ] S11 更新ログ ― 先頭行に本日分が追記されているか
```
